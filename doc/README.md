# Документація EZ MONITOR

Ця директорія - робоча база знань про продукт, архітектуру, PTS-2, MVP, інтеграції та процес розробки. Vendor-файли лежать окремо в `vendor/` і не редагуються без прямого запиту.

## Як користуватися

- Починай із `repository-analysis.md`, якщо треба швидко зрозуміти стан репозиторію.
- Продуктові межі й бізнес-вимоги живуть у `product/` та `product_requirements/`.
- Архітектурні схеми, флоу й моделі живуть у `architecture/`, а рішення - в `adr/`.
- API-контракти живуть у `api/`, БД - у `db/`, доменні правила - у `domain/`.
- План робіт і беклог живуть у `tasks/`.
- PTS-2 дослідження та чорнові нотатки живуть у `notes/pts2/`.

## Структура

| Директорія | Для чого |
| --- | --- |
| `_templates/` | Шаблони ADR, специфікацій фіч, задач і дослідницьких нотаток. |
| `adr/` | Архітектурні рішення, які впливають на майбутню розробку. |
| `api/` | REST/OpenAPI/jsonPTS контракти, приклади запитів і відповідей. |
| `architecture/` | Системний дизайн, ingestion, AI-доступ, RBAC, deployment-рішення. |
| `db/` | ERD, нотатки по міграціях, seed strategy, правила якості даних. |
| `dev/` | Локальний запуск, інструменти, CI, troubleshooting. |
| `domain/` | Бізнес-словник: АЗС, ПРК, резервуари, картки, alerts. |
| `guides/` | Практичні інструкції для розробки й дослідження PTS-2. |
| `notes/` | Дослідницькі нотатки, висновки, відкриті питання. |
| `owner/` | Рішення власника продукту, бізнес-пріоритети, обмеження. |
| `product/` | Бачення продукту, personas, ролі користувачів, користувацькі сценарії. |
| `product_requirements/` | PRD/MVP-вимоги, критерії приймання. |
| `security/` | Модель загроз, RBAC, секрети, аудит, AI-доступ до даних. |
| `spec/` | Технічні специфікації фіч перед реалізацією. |
| `tasks/` | Беклог, milestones, робочі чеклісти. |
| `ui/` | UX flows, design tokens, нотатки по екранах, verification. |
